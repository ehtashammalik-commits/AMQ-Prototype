# Phase 3.1: ExpertFlow API Integration - Push Transcriptions to CIM

## 🎯 Objective

After transcription is generated, allow users to **selectively push individual transcription messages** to ExpertFlow's Conversation Information Manager (CIM) API as either Agent or Customer messages.

---

## 📋 Requirements

### **1. Conversation ID Management**
- ✅ **Two Options:**
  - **Auto-Generate**: Generate unique `CONVERSATION_ID` for each audio file
  - **Manual Input**: Allow users to input existing Conversation ID from ExpertFlow
- Store conversation ID with the audio file metadata
- Display conversation ID prominently in the UI
- Allow users to edit/change conversation ID anytime before pushing to CIM
- **One-to-One Relationship**: Each audio file has exactly ONE Conversation ID
- Show which Conversation ID will be used when pushing messages

### **2. Message Push Button**
- ✅ Add a **"Push to CIM"** button on each transcription message
- Button appears after transcription is generated
- Shows which type the message will be pushed as (Agent/Customer)
- Color-coded: Blue for Agent, Green for Customer

### **3. Message Type Selection**
- ✅ Popup/dropdown to select message sender type when pushing:
  - **Agent Message** - Message from support agent
  - **Customer Message** - Message from customer
- Remember last selected type to reduce clicks
- Show preview of payload before sending

### **4. ExpertFlow API Integration**
- ✅ POST to: `https://missouri.expertflow.com/conversation-manager/customer-topics/{CONVERSATION_ID}/events`
- Send formatted payload based on message type
- Handle API authentication (Bearer token from config)
- Show success/error feedback to user

## 🏗️ Implementation Plan

### **Step 1: Data Models**
```typescript
// conversation.model.ts
export interface Conversation {
  conversationId: string;        // ExpertFlow Conversation ID (user-provided or auto-generated)
  audioFileId: string;           // Local audio file ID
  audioFileName: string;
  createdAt: Date;
  
  // Optional: Agent/Customer info for CIM
  agentName: string;
  agentId: string;
  customerName: string;
  customerId: string;
  
  // ExpertFlow API credentials
  expertFlowToken?: string;      // Bearer token
  expertFlowEndpoint?: string;   // Full API endpoint
  
  // Metadata
  isAutoGenerated: boolean;      // Was conversation ID auto-generated?
  lastModified: Date;
}

export interface TranscriptionMessage {
  id: string;
  speaker: 'agent' | 'customer';
  text: string;
  timestamp: number;
  conversationId: string;        // Reference to parent conversation
  
  // Push to CIM tracking
  pushed: boolean;               // Has this been pushed to CIM?
  pushedAt?: Date;
  pushedAs?: 'AGENT_MESSAGE' | 'CUSTOMER_MESSAGE';  // Which type was it pushed as?
  pushStatus?: 'pending' | 'success' | 'error';
  pushError?: string;
  
  // Retry tracking
  pushRetries: number;
  lastPushAttempt?: Date;
}

export interface CIMPushLog {
  messageId: string;
  conversationId: string;
  timestamp: Date;
  status: 'success' | 'error';
  messageType: 'AGENT_MESSAGE' | 'CUSTOMER_MESSAGE';
  apiResponse?: any;
  errorDetails?: string;
}
```

### **Step 4: Workflow**

1. **User uploads audio file**
   ↓
2. **System generates unique Conversation ID** 
   - Auto-generated UUID shown in UI
   - User can see: "🆔 Conversation ID: 6a4e036eb0c4a8cd4d2dc261"
   ↓
3. **User has 3 options:**
   
   **Option A: Use Auto-Generated ID**
   - Click ✅ "Use This ID"
   - Proceed to transcription
   
   **Option B: Use Existing ExpertFlow ID**
   - Click ✏️ "Edit"
   - Paste existing Conversation ID from ExpertFlow
   - Confirm
   
   **Option C: Generate New ID**
   - Click 🔄 "Regenerate"
   - New ID created
   ↓
4. **User transcribes audio** (with Conversation ID stored)
   ↓
5. **Each transcription message displays:**
   - Speaker type (Agent/Customer)
   - Message text
   - 📤 "Push to CIM" button
   ↓
6. **User clicks "📤 Push to CIM" on a message**
   ↓
7. **Confirmation dialog appears:**
   ```
   Push to ExpertFlow CIM?
   
   Conversation ID: 6a4e036eb0c4a8cd4d2dc261
   Message Type: ☐ Agent  ☑ Customer
   Text: "بخير شكراً، لدي مشكلة..."
   
   [Cancel]  [Push to CIM]
   ```
   ↓
8. **System builds payload with:**
   - Conversation ID (from step 3)
   - Message text
   - Sender type (confirmed by user)
   - Sender info (agent/customer details)
   - Timestamps (ISO format)
   - GUIDs for each event
   ↓
9. **POST to ExpertFlow API:**
   ```
   POST https://missouri.expertflow.com/conversation-manager/customer-topics/
       {conversationId}/events
   ```
   ↓
10. **Show result:**
    - ✓ **Success** 
      - Green checkmark on message
      - Show: "Pushed at 2:35 PM"
      - Button becomes disabled
      - Log entry created in push history
    
    - ✗ **Error** 
      - Red warning icon
      - Show error message
      - Retry button appears
      - Log error for debugging
